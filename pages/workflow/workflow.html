WORKFLOW

Step 1

Study Area & Data Preparation

Our group worked on the assigned country by first selecting its boundary from the global
administrative vector dataset using QGIS's "Select by attribute" tool.
Environmental and demographic datasets used include:
  • CAMS monthly raster data (PM2.5, PM10, NO₂ for 2013–2022)
  • ESA CCI Land Cover 2022 (NetCDF)
  • WorldPop 2020 population raster
All data were reprojected to WGS84 and clipped to the study area using Clip Raster by Mask.
Administrative boundaries of Spain

Step 2

Pollution Raster Aggregation

For each pollutant, we calculated:
  • December 2022 raster
  • Yearly average of 2022
  • 5-year average (2017–2021)
  • 2022 minus 5-year mean (to analyze trend)
These operations were done using QGIS’s Raster Calculator, r.series, and conditional expression tools.
All rasters were reclassified into 5 concentration classes according to EU thresholds, which simplifies later combination with other variables.
EU concentration levels

Step 3

Land Cover Reclassification

ESA Land Cover data was simplified from 22 classes to 6 categories (IPCC standard) using the “Reclassify by table” tool.
This makes it easier to visually interpret the dominant land use types and relate them to pollution patterns.
Legend simplification using IPCC classes (based on ESA CCI LC 2022)

Step 4

Population Classification

The WorldPop raster was classified into 5 quantile-based classes using r.quantiles and Reclassify by rules.
This allows meaningful comparison with pollution levels in different density zones.
Population Density Classes – 2020

Step 5

Bivariate Indicator Calculation

We calculated bivariate indicators combining pollution class (max per zone) and population class (median per zone) at Admin Level 2.
The formula used was:
Bivariate Code = Pollution_Class × 10 + Population_Class
Resulting values (e.g. 55 = high pol + high pop) were mapped using a 5×5 color matrix.
combined population and PM2.5 pollution

Step 6

Output and Integration

Key results (pollution trends, reclassified maps, bivariate indicators) were exported from QGIS in PNG format and integrated into the final WebGIS page using Leaflet’s L.imageOverlay().
Some additional visualizations such as pie charts or legends were created using QGIS Layout Manager or DataPlotly.
